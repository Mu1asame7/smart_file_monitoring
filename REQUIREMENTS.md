# Система умного мониторинга файлов

## Техническое задание

### 1. Цель проекта

*Опиши одной-двумя фразами, зачем нужен этот проект.*

Пример: "Автоматизация организации файлов в системе с возможностью поиска по содержимому изображений."

### 2. Функциональные требования

#### 2.1. Отслеживание файлов

- [ ] Какие папки отслеживать? 
    - C:\Users\PC\Desktop, C:\Users\PC\Downloads
- [ ] Следить за: созданием, изменением, удалением, перемещением? 
    -Добавить поле status в БД: 'active', 'deleted', 'moved'. Заполняется при любом действии с файлом.
    -При создании добавлять в БД данные, сортировать по критериям и переместить.  
    -При изменении меняются свойства файла, необходимо изменить данные в БД о нем.
    -При удалении можно оставить в БД только те свойства файла, которые помогут инициализировать его при повторном скачивании (Например: Наименование, Размер, Хэшсумма). При поиске не отображаются файлы со свойством status == 'deleted'
    -При перемещениинеобходимо добавить в бд новый путь к файлу, чтобы была информация о его местонахождении
- [ ] Как часто проверять?
    -Изначально по расписанию. В дальнейшем в режиме реального времени -> происходит одно из перечисленных выше действий, запускается программа

#### 2.2. Сортировка файлов

- [ ] По каким категориям сортировать? 
    -формат(Изображения, документы, видео, архивы, музыка, исполняемые файлы)
    -по дате(необходимо создавать директории дат формата Год/Месяц/День в конечных папках)
    -Исполняемые файлы просто сортируются
    -Музыка (.mp3, .flac, .wav) - необходимо извлчечь метаданные (исполнитель, альбом)
- [ ] Куда перемещать? 
    -в отдельную структуру
- [ ] Что делать с дубликатами? 
    -Дубликаты необходимо перемещать в отдельную директорию C:\Duplicates\
    -Также создаются отдельные директории по датам формата Год/Месяц/День(Дата удаления) 
    -Через неделю происходит удаление файлов из директории даты. При первоначальной настройке уведомлять пользователя об удалении
    -Приходит системное уведомление о том, что "Сегодня в "Время" произойдет удаление дубликатов файлов в директории "Путь к директории""

#### 2.3. Индексация и поиск

- [ ] Какие данные индексировать? 
    -имя, путь, размер, дата, содержимое
- [ ] Какие типы файлов обрабатывать для индексации?
    -Текстовые файлы (.txt, .py, .json, .xml) — индексируем содержимое полностью
    -Документы (.pdf, .docx) — нужно извлекать текст специальными библиотеками
    -Изображения — только через OCR (который мы сделаем позже)
    -Архивы — индексируем только список файлов внутри
- [ ] Как организовать поиск? 
    -по имени, по содержимому, по дате

#### 2.4. OCR для изображений

- [ ] Для каких форматов изображений делать OCR? 
    -.jpg, .jpeg, .png, .gif, .bmp, .tiff
- [ ] Какие языки распознавать? 
    -Русский
- [ ] Что делать с результатом распознавания? 
    -Добавлять информацию о картинке в БД

### 3. Нефункциональные требования

- [ ] **Производительность:** Как быстро должна работать программа? (Проект создается для собственного использования. Нет ограничения на количество файлов в папке, так как файлов не будет огромное количетсво. Файл по поиску должен находиться до 1 секунды. Максимальное количетсво файлов в БД 10к)
- [ ] **Надежность:** Что делать при сбоях? (Если программа упала, необходимо записать лог ошибки в файл и попытаться перезапустить. Уведомить, если на диске с отсортированными файлами осталось менее 100 мб свободного места всплывающим окном. Если файл заблокирован в другой программе, пропустить его обработку и попытаться обработать его ещё раз при следующей обработке)
- [ ] **Удобство:** Должен ли быть интерфейс? (консольный интерфейс. Добавить веб-интерфейс после разработки)
- [ ] **Переносимость:** Должна ли работать на Windows/Linux/Mac? (На Windows и Linux/Mac)

### 4. Ограничения и допущения

- Какие типы файлов НЕ обрабатываем? 
    -Системные файлы (.dll, .sys, .tmp) 
    -Временные файлы браузеров
- Какие папки НЕ трогаем? 
    -Системные папки Windows (C:\Windows, C:\Program Files) 
    -Скрытые папки (начинающиеся с .)
- Что делаем с очень большими файлами? 
    -Файлы >1ГБ происходит частичная обработка -> Не индексируем содержимое, только перемещаем
